Table 4.13   Use Case Description for <UC011 Validate Image Content>
================================================================================

Use Case ID         | UC011
Use Case Name       | Validate Image Content
Actors              | Vision Transformer AI System, Patient (indirect)
Description         | How the system automatically checks uploaded images for
                    | inappropriate content (NSFW filter) and verifies minimum 25%
                    | skin content before allowing medical analysis.
Pre-conditions      | 1. Image has been uploaded or captured (UC009 or UC010)
                    | 2. Image file is in valid format
                    | 3. NSFW detection model is loaded and operational
Post-conditions     | 1. Image is validated as appropriate for medical analysis
                    | 2. Image contains sufficient skin content (≥25%)
                    | 3. Validation result is logged in database
                    | 4. Image proceeds to AI lesion analysis (UC014)
Normal Flow (NF)    | 1. System receives uploaded image from UC009 or UC010
                    | 2. System converts image to RGB format if needed
                    | 3. System preprocesses image to 224x224 pixels
                    | 4. System calculates skin percentage using 4 criteria:
                    |    a. RGB channel ordering (R > G > B)
                    |    b. RGB value ranges (skin tone ranges)
                    |    c. Brightness range (20-90%)
                    |    d. Saturation range (10-70%)
                    | 5. System counts pixels passing ≥3 criteria as "skin-like"
                    | 6. System calculates total skin percentage
                    | 7. System checks if skin percentage ≥ 25% threshold
                    | 8. System performs text/graphics detection:
                    |    - Edge density analysis
                    |    - High contrast detection
                    |    - Pure color block detection
                    | 9. System calculates NSFW score using heuristic
                    | 10. System checks NSFW score < 0.35 threshold
                    | 11. All validation checks pass
                    | 12. System logs "Validation passed" with skin percentage
                    | 13. System returns success status
                    | 14. Image proceeds to UC014 (AI Lesion Analysis)
Alternative Flow    | None (Automatic system process)
(AF)                | 
Exception Flow (EF) | EF1: Insufficient Skin Content
                    | 1. At step 7, skin percentage < 25%
                    | 2. Go to UC012 (Reject Non-Skin Images)
                    | 
                    | EF2: Text/Graphics Detected
                    | 1. At step 8, high edge density or contrast detected
                    | 2. System identifies poster, document, or screenshot
                    | 3. Go to UC012 (Reject Non-Skin Images)
                    | 
                    | EF3: NSFW Content Detected
                    | 1. At step 10, NSFW score exceeds 0.35 threshold
                    | 2. System logs "NSFW content detected"
                    | 3. System rejects image with generic error message
                    | 4. Display "Inappropriate content detected" to patient
                    | 5. Patient must upload different image
