UPDATED PARAGRAPH (Your SkinGuard system):
------------------------------------------
This complete package diagram illustrates the architectural organization of the 
SkinGuard system into four main layers: Presentation Layer (frontend user 
interface), Application Layer (backend business logic), Data Layer (database 
storage), and External Services Layer (third-party integrations). The 
Presentation Layer consists of user interface components including authentication 
pages, patient dashboard, hospital staff dashboard, image upload and capture 
module, symptom wizard, analysis results display, hospital search with Google 
Maps integration, appointment booking interface, and report review interface for 
hospital staff. The Application Layer implements RESTful API endpoints for 
authentication, patient management, AI analysis, appointment scheduling, and 
hospital search, along with business logic modules for user management, NSFW 
image validation enforcing 25% skin threshold, AI analysis service integrating 
Vision Transformer model, risk assessment with conservative bias for melanoma 
cases, appointment manager with privacy protection, JWT authentication, and audit 
logging for HIPAA compliance. The Data Layer utilizes PostgreSQL with seven 
normalized tables (profiles, patient_data, doctors, medical_reports, 
appointments, reviews, audit_logs) organized into user data, analysis data, 
appointment data, and system data packages. The External Services Layer 
integrates Vision Transformer AI from Hugging Face for 7-class skin cancer 
classification with 84% accuracy, Google Maps API for hospital search within 50km 
radius, and SMTP email server for appointment confirmations and notifications.

DETAILED VERSION (Academic Style):
-----------------------------------
Figure X.X presents the complete package diagram for the SkinGuard AI-Powered 
Skin Cancer Detection System, illustrating the system's architectural 
organization into four distinct layers that demonstrate separation of concerns 
and modular design principles. The Presentation Layer, implemented using React 
framework, is decomposed into three primary packages: the User Interface package 
containing authentication pages for login and registration, patient dashboard for 
viewing analysis history and managing appointments, and hospital staff dashboard 
for reviewing anonymized reports and confirming appointments; the Core Features 
package encompassing image upload with camera capture functionality, four-step 
symptom wizard for collecting clinical context (body location, sensations, visual 
changes, duration), and analysis results display showing AI predictions with 
color-coded risk levels (red for URGENT, orange for HIGH, yellow for MEDIUM, 
green for LOW); and the Hospital Features package providing Google Maps-
integrated hospital search within 50km radius, appointment booking workflow with 
privacy protection, and report review interface enabling hospital staff to 
evaluate patient submissions while maintaining anonymity until appointment 
confirmation.

The Application Layer, developed using FastAPI backend framework, is organized 
into four packages that implement the system's business logic and coordinate data 
flow between the frontend, database, and external services. The API Endpoints 
package provides RESTful routes for authentication (POST /auth/login, POST /auth/
register), patient operations (GET /patients/profile, PUT /patients/profile), 
analysis submission (POST /analysis/upload, POST /analysis/symptom-wizard), 
appointment management (POST /appointments, PUT /appointments/confirm), and 
hospital search (GET /hospitals/search). The Business Logic package contains 
critical modules including User Management for authentication with JWT tokens and 
role-based access control (patient, doctor), Image Validation implementing NSFW 
filter with 4-criteria algorithm enforcing minimum 25% skin content to reject 
posters and distant shots, AI Analysis Service integrating Vision Transformer 
(ViT-H-14) model from Hugging Face to classify lesions into seven cancer types 
(melanoma, basal cell carcinoma, squamous cell carcinoma, actinic keratosis, 
benign keratosis, dermatofibroma, vascular lesion) with 84% accuracy, Risk 
Assessment applying conservative bias to elevate borderline melanoma cases (80-
84% confidence) to URGENT classification for patient safety, and Appointment 
Manager orchestrating the booking workflow with privacy-by-design revealing 
patient identity only after hospital staff confirmation. The External Integration 
package provides client interfaces for Vision Transformer AI (Hugging Face API), 
Google Maps API for geolocation-based hospital search sorting results by distance 
and rating, and SMTP Email Service for appointment confirmations, rejections, and 
reminder notifications. The Security & Logging package implements cross-cutting 
concerns including JWT Authentication with 30-minute access tokens and 7-day 
refresh tokens, Privacy Protection enforcing anonymized report presentation until 
explicit consent, and Audit Logger recording all data access events 
(user_registration, report_created, identity_revealed, appointment_confirmed) for 
HIPAA compliance and forensic analysis.

The Data Layer utilizes PostgreSQL relational database management system with 
seven normalized tables achieving Third Normal Form (3NF) to eliminate data 
redundancy and ensure referential integrity through foreign key constraints. The 
User Data package contains profiles table storing authentication credentials 
(email, password hash) and basic user information (full name, role, verified 
status) for all users regardless of role, patient_data table extending profiles 
with health-specific information (age for melanoma risk factors, Fitzpatrick 
skin_type I-VI classification, family_history of hereditary cancer), and doctors 
table extending profiles with professional credentials (license_no, 
specialization) and clinic information (clinic_name, geographic coordinates for 
Google Maps integration, whatsapp_no for patient communication, average_rating 
and review_count aggregated from reviews). The Analysis Data package includes 
medical_reports table storing complete AI workflow output (image_url pointing to 
cloud storage, ai_prediction JSONB field containing probabilities for all seven 
cancer types, symptoms text from 8-question wizard, risk_level enum driving 
appointment urgency, body_location, consultation_notes added by reviewing 
doctors) with timestamps for audit trails. The Appointment Data package manages 
appointments table linking patients with doctors through foreign keys 
(patient_id, doctor_id, optional report_id), tracking appointment lifecycle 
(pending, confirmed, rejected, completed, cancelled) with status transitions 
triggering identity reveal mechanism and audit log entries, supporting both in-
person and video_call consultation types, and reviews table implementing doctor 
rating system with 1-5 star ratings, optional qualitative review_text, and 
flagged boolean for content moderation. The System Data package contains 
audit_logs table providing comprehensive activity tracking (log_id, user_id, 
action, resource_type, resource_id, metadata JSONB) for security incident 
investigation and regulatory compliance verification.

The External Services Layer integrates three critical third-party systems through 
well-defined client interfaces that abstract implementation details and enable 
testability through dependency injection. The Vision Transformer AI service, 
hosted on Hugging Face model hub, performs deep learning-based image 
classification using the ViT-H-14 architecture pre-trained on dermoscopic lesion 
datasets, accepting preprocessed images validated by the NSFW filter and 
returning probability distributions across seven cancer types with confidence 
scores that feed into the risk assessment algorithm. The Google Maps API service 
provides two primary functions: Places API for searching hospitals and 
dermatology clinics within 50km radius of patient location using geographic 
coordinates (lat, lng) and returning up to 180 results sorted by distance and 
average_rating, and Directions API for calculating estimated travel time and 
route guidance to selected hospitals. The SMTP Email Server enables automated 
notification delivery for appointment workflow events including email verification 
during registration, appointment confirmation emails sent to patients after 
hospital staff approval (triggering patient identity reveal), appointment 
rejection emails maintaining patient anonymity, and reminder emails for upcoming 
consultations scheduled within 48 hours.

The package diagram demonstrates adherence to software engineering best practices 
including separation of concerns through layered architecture, modular design 
enabling independent development and testing of packages, loose coupling between 
layers enforced through well-defined interfaces (REST API, database ORM, external 
service clients), high cohesion within packages grouping related functionality, 
scalability through horizontal scaling of stateless Application Layer components, 
maintainability allowing changes to one package without cascading effects on 
others, and testability supporting unit tests for individual packages and 
integration tests for inter-package dependencies. Cross-cutting concerns such as 
authentication (JWT tokens), privacy protection (anonymization until consent), 
and audit logging (HIPAA compliance) are implemented as dedicated packages that 
interact with multiple layers, ensuring consistent enforcement of security and 
regulatory requirements throughout the system.
