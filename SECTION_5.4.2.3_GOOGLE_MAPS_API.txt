5.4.2.3 Google Maps API Handler

The Google Maps API handler was tested by tracing the try/except blocks, empty 
result conditionals, and parameter validation logic. The handler constructs 
different queries depending on the user's location, search radius, and filter 
preferences, so each branch was verified. Testing covered location geocoding 
(converting city names to coordinates), radius filtering (50km default), 
hospital type filtering (dermatology specialty vs general hospitals), rating-
based sorting (4.5+ stars prioritized), result pagination (up to 180 hospitals), 
error handling for API timeouts and invalid API keys, and GPS permission denial 
fallback to manual location entry. The handler implements retry logic with 
exponential backoff when API rate limits are exceeded and caches successful 
search results in sessionStorage to avoid redundant API calls during the same 
session.

GOOGLE MAPS API INTEGRATION DETAILS
================================================================================

Your system integrates THREE Google Maps APIs:

1. PLACES API (Nearby Search):
   - Endpoint: /maps/api/place/nearbysearch/json
   - Parameters: location, radius, type, keyword
   - Returns: Array of hospitals with basic info
   - Cost: $0.032 per request (first 100k/month free)

2. GEOCODING API:
   - Endpoint: /maps/api/geocode/json
   - Purpose: Convert "Kuala Lumpur" → lat/lng coordinates
   - Fallback: When GPS denied or unavailable
   - Cost: $0.005 per request

3. PLACE DETAILS API:
   - Endpoint: /maps/api/place/details/json
   - Purpose: Get hours, reviews, photos for selected hospital
   - Triggered: User clicks "View Details" button
   - Cost: $0.017 per request

TOTAL API COSTS OPTIMIZED:
- Session caching reduces redundant Nearby Search calls
- Lazy loading of Place Details (only when user requests)
- Batch geocoding for common cities (pre-cached coordinates)
