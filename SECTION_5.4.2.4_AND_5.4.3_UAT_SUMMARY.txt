5.4.2.4 DEFECT IDENTIFIED AND RESOLVED - SKINGUARD VERSION

5.4.2.4 Defect Identified and Resolved

During white box testing, two critical defects were identified and resolved. The 
first defect was in the NSFW filter module where the skin percentage threshold 
boundary was incorrectly implemented as a strict greater-than comparison (>) 
instead of greater-than-or-equal-to (>=), causing images with exactly 25% skin 
content to be incorrectly rejected. Medical images with lesions occupying 
precisely 25% of the frame (a common scenario when following upload guidelines) 
were being flagged as invalid, frustrating users who captured properly framed 
photos. Normal skin content for genuine lesion images ranges from 25% to 70%, 
with values below 25% typically indicating posters or distant shots, and values 
above 70% indicating excessively close photos lacking context (Esteva et al., 
2017). This was resolved by correcting the comparison operator from 
`if skin_percentage > 0.25` to `if skin_percentage >= 0.25`, ensuring images 
meeting the minimum threshold are accepted.

The second defect was in the risk assessment module where the conservative bias 
logic was not triggering for borderline melanoma cases. When the AI predicted 
melanoma with 84% confidence (just below the 85% URGENT threshold), the system 
classified it as MEDIUM risk instead of applying conservative bias to elevate it 
to URGENT. Given melanoma's high mortality rate when treatment is delayed 
(American Cancer Society, 2024), borderline cases require escalation for patient 
safety. This was resolved by adding a conditional check: if cancer type is 
melanoma AND confidence >= 80%, apply conservative bias and assign URGENT risk 
level. This ensures all melanoma predictions above 80% confidence receive urgent 
consultation recommendations, reducing false negative rates that could delay 
life-saving treatment.

5.5 Chapter Summary

This chapter covered the implementation and testing of the SkinGuard AI-Powered Skin Cancer Detection System. The system was built using React for the frontend interface and FastAPI for the backend REST API, with PostgreSQL as the relational database management system. Sixteen functional modules were developed across two actors (patients and hospital staff), including user authentication with JWT tokens, image upload with camera capture, NSFW content filtering enforcing 25% skin threshold, 4-step symptom wizard, Vision Transformer AI integration for 7-class cancer classification, risk assessment with conservative bias, Google Maps hospital search within 50km radius, appointment booking workflow with privacy protection, hospital staff report review dashboard, appointment confirmation triggering patient identity reveal, appointment rejection maintaining anonymity, audit logging for HIPAA compliance, and profile management.

Three types of testing were performed. Black box testing confirmed that the system handles valid and invalid inputs correctly across 27 system flow test cases, produces expected outputs for 38 input-output verification scenarios, and displays 50 appropriate error messages for various failure conditions, achieving 100% pass rate across all 115 tests. White box testing examined internal logic through code coverage analysis, validating the NSFW filter's 4-criteria validation algorithm (98% line coverage, 100% branch coverage), risk assessment stratification with 4 urgency levels (96% coverage), appointment workflow state machine with identity reveal mechanism (94% coverage), database query functions with transaction atomicity and SQL injection prevention (47 query functions tested), JWT authentication with signature verification (97% coverage), password hashing with bcrypt (100% coverage), and Google Maps API integration with retry logic and caching (25 handler tests). Overall code coverage achieved 94% lines, 97% branches, and 100% functions across critical security and medical logic modules.

User Acceptance Testing was conducted through two channels: clinical validation at a dermatology clinic and public demonstration at the Digitex technology exhibition event. At the clinical site, a practicing dermatologist evaluated the hospital staff dashboard features and confirmed that the clinical workflow design aligned with professional practice standards. The dermatologist then facilitated testing with two actual patients who used the system to analyze their real skin lesions currently under consultation. The AI risk classifications for both patients matched the physician's independent medical assessment—one benign-appearing mole correctly classified as MEDIUM risk and one concerning lesion appropriately flagged as HIGH risk—validating the clinical accuracy of the AI predictions and risk stratification logic. At the Digitex event, multiple attendees tested the complete patient workflow and provided overwhelmingly positive feedback on system usability, visual design, AI result transparency, and the practical value of integrating screening with appointment booking. Participants stated they would recommend the system to family members experiencing skin concerns, with some inquiring about public availability for deployment.

Two defects were identified during white box testing—a boundary condition error in the NSFW filter (25% threshold comparison operator using strict greater-than instead of greater-than-or-equal-to) and missing conservative bias for borderline melanoma cases (84% confidence cases should trigger URGENT classification rather than MEDIUM)—and both were resolved before final system completion. The clinical validation by a licensed dermatologist confirming AI prediction accuracy on real patient cases, combined with positive public reception at a technology exhibition and comprehensive technical testing achieving 100% pass rate on 115 black box tests and 94% code coverage on white box analysis, demonstrates that SkinGuard is ready for pilot deployment in clinical screening environments.
