DESCRIPTION FOR THESIS
================================================================================

This package diagram describes the skin cancer detection and AI analysis
functionality. In the package diagram, the Model includes Medical Report,
Symptom Response, Image Validation, and Risk Assessment classes. View
consists of the image upload page, image capture page, symptom wizard page
(4-step questionnaire), analysis results page, and report history page. For
controllers, the Image Upload Controller, Image Validation Controller (NSFW
filter with 25% skin threshold and poster detection), AI Analysis Controller
(Vision Transformer integration), Risk Assessment Controller (stratification
with conservative bias), and Report Management Controller are included.

The Skin Detection Module implements a multi-stage AI analysis pipeline
following the MVC pattern. The View layer provides user interfaces for image
input (upload or camera capture), a four-step symptom wizard collecting body
location, sensations (itching, pain, burning), visual changes (color, size,
shape), and duration, and a comprehensive results display showing risk level
banner, AI classification with confidence scores for all seven cancer types,
lesion localization, and recommended consultation timelines.

The Controller layer orchestrates the analysis workflow through five
specialized controllers: Image Upload Controller validates file format and
size (max 10MB, JPEG/PNG), Image Validation Controller enforces content
safety through NSFW filtering (checking skin percentage ≥25%, detecting text
or graphics indicative of posters or documents, and screening explicit
content), AI Analysis Controller interfaces with the Vision Transformer AI
model from Hugging Face (ViT-H-14 architecture, 84% accuracy on HAM10000
dataset, classifying into seven cancer types), Risk Assessment Controller
applies medical decision rules (URGENT for melanoma or confidence ≥85%, HIGH
for basal cell carcinoma or actinic keratoses, MEDIUM for confidence 60-85%,
LOW otherwise) with conservative bias favoring false positives over false
negatives for patient safety, and Report Management Controller handles report
storage, retrieval, PDF generation, and sharing with hospital staff.

The Model layer persists data through four database tables: medical_reports
(storing AI predictions, confidence scores, risk levels, body locations),
symptom_responses (structured symptom data from wizard), image_validation
(validation metrics including skin percentage, NSFW scores, rejection
reasons), and risk_assessment (urgency scores, justification text, timeline
recommendations). External integration with Vision Transformer AI via Hugging
Face API enables real-time skin lesion classification, while image storage
services preserve both original uploads and thumbnails for performance
optimization. The system logs all validation attempts to audit_logs for
quality monitoring and continuous improvement of the NSFW filter thresholds.
