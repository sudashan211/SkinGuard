5.4.1.3 Error Messages

Error message testing verifies that the system returns clear and appropriate 
messages when invalid inputs are submitted or unexpected actions are performed 
(Tan et al., 2022). This ensures users are properly guided when errors occur 
rather than the system failing silently. Testing covered authentication errors 
with empty fields or invalid credentials, image upload errors with incorrect 
formats or oversized files, NSFW validation failures with specific rejection 
reasons, symptom wizard validation errors for incomplete questionnaires, AI 
analysis errors when model inference fails, hospital search errors when Google 
Maps API is unavailable, appointment booking errors with invalid date 
selections, email notification failures with SMTP connection issues, and 
profile management errors with weak passwords or invalid data formats. All 
error messages were designed to be user-friendly, actionable, and 
security-conscious (avoiding information leakage about system internals).
